# 组合风险卡 — 功能规格

> 本文只描述**功能与数据**,不含任何设计/视觉决定(布局、配色、层级由设计负责)。

## 目标

把整块推荐板聚合成**一个账户级风险视图**,回答逐张推荐卡看不到的问题:
"如果把今天板上这些仓位都开了,我的**净敞口**是什么,**尾部**会在哪里一起爆?"

定位:今日整个推荐板的风险体检,不属于任何单个标的,是全局摘要。

## 数据来源

Dashboard API 响应新增字段 `bookRisk`(后端已实现,数据现成)。前端只绑数据,不做业务计算(除下方明确的派生值)。

## 字段定义

| 字段 | 类型 | 单位 / 含义 |
|---|---|---|
| `positions` | number | 板上仓位数 |
| `shortVolCount` | number | 其中卖波动(净空 vega)的仓位数 |
| `netDelta` | number | 净 delta(**每仓位 1 张合约**的原始 greek 求和) |
| `netGamma` | number | 净 gamma(同上) |
| `netVega` | number | 净 vega(同上) |
| `netTheta` | number | 净 theta(同上) |
| `aggMaxLossUsd` | number | 同时打满最大亏损(**美元**,已 ×100,仅统计定义风险仓位) |
| `netThetaUsd` | number | 每日 carry(**美元**,= netTheta × 100) |
| `undefinedRiskCount` | number | 无定义风险(尾部无封顶)的仓位数 |
| `flags` | string[] | 人话尾部提示,已按重要性排序,可直接逐条渲染 |

### 单位口径(重要)

- 四个 `net*`(delta/gamma/vega/theta)是**每仓位 1 张的原始 greek 求和** —— 表达**符号与相对大小**,不是美元。
- `aggMaxLossUsd` / `netThetaUsd` 是**美元**(已 ×100)。
- 整卡是"单位仓位下的风险姿态",**不是用户真实账户的美元敞口**(券商实际持仓当前拉不到)。这一点需要让用户明确知道。

## 需要呈现的信息(功能层面,不规定"怎么摆")

1. **板子构成**:共几个仓位,其中几个是卖波动(`positions`、`shortVolCount`)。
2. **收益 vs 尾部的对比**:每日 carry(`netThetaUsd`)与同时打满最大亏损(`aggMaxLossUsd`),并让用户直观感受到两者的**不对称**(见派生值)。这是本卡最核心要传达的信息。
3. **四个净敞口**:`netDelta / netGamma / netVega / netTheta`,并对每个标注其风险含义:
   - delta → 方向是否中性
   - gamma → 是否净空(净空 = 跳空时亏损加速)
   - vega → 是否净空(净空 = 全市场 IV 上冲的尾部主力)
   - theta → carry 来源(卖方通常为正)
4. **尾部提示**:逐条展示 `flags[]`。

## 计算 / 派生逻辑(前端算)

- **不对称倍数** = `aggMaxLossUsd / netThetaUsd`,取整。用于表达"每天挣一点、尾部一次亏很多"。
- **delta 姿态**:`|netDelta| ≤ 1` → 中性;`netDelta > 1` → 偏多;`netDelta < −1` → 偏空(阈值与后端 flag 一致)。
- 所有屏显数字必须做取整/定小数(金额取整,greeks 2 位小数),避免浮点尾巴。

## 状态与边界

| 状态 | 行为 |
|---|---|
| `positions === 0`(空板) | 收起主体,仅显示空状态「今日无推荐」 |
| `netThetaUsd ≤ 0`(极罕见) | 隐藏"不对称倍数"(不可除零) |
| `undefinedRiskCount > 0` | 在最大亏损旁提示 `+{undefinedRiskCount} 仓位无封顶`(尾部实际更糟) |
| `flags` 为空 | 隐藏尾部提示区(或显示「无显著集中风险」) |
| `positions === 1` | 相关性→1 那条 flag 不会出现(后端已按 n≥2 控制),正常渲染 |

## 交互

- **v1:纯展示,静态,无交互。**
- v2(暂不做,仅备忘):点某个 greek → 展开该敞口由哪些标的贡献;点最大亏损 → 各仓位最大亏损明细。

## 备注

- `flags[]` 已是成品中文句子(后端 `summarizeBookRisk` 生成),内容示例:
  - `净空 vega N → 全市场 IV 上冲会同时打击整个篮子(尾部同向)`
  - `净空 gamma N → 标的跳空时亏损加速(短 gamma 的压路机风险)`
  - `M 个仓位全是卖波动、且都在相关大盘股 → "分散"是假象,崩盘时相关性→1 一起爆`
- 定稿后前端工作量:绑 `bookRisk` 字段 + 两个派生值 + 状态判断,**不需要改后端**。
